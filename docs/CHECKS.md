# Stage-A check catalog

Every deterministic gate a One Pond artifact must clear before a reviewer (Stage B) or the
Gatekeeper ever sees it. A check runs in Stage A **only after it certifies** — green on every
good fixture and red on every bad fixture (`harness/checks/registry.py`); an uncertified check
is inert and can neither wave bad work through nor wedge the loop. Checks run cheapest tier
first (`STATIC → STRUCTURAL → DYNAMIC`) and Stage A **fail-fasts** on the first FAIL, so an
expensive simulation never runs behind a cheap syntax reject.

A check returns **SKIP** when it does not apply (no target files, or an opt-in mechanic the
pond didn't use); `stage_a_passed` requires at least one non-SKIP check to have run and none to
have FAILed. Numeric `metrics` a check emits are minted as **monotonic ratchet floors** (keyed
per ticket), so an accepted artifact can never be silently re-accepted below the quality it
first passed at.

This catalog is generated by hand from the code; if it drifts, the code
(`harness/checks/`, `game/onepond/checks.py`) is the source of truth.

## Harness checks (`default_registry`, apply to any ticket)

| id | tier | gates | metric → floor |
|----|------|-------|-----------|
| `non_empty_artifact` | STATIC | at least one non-empty file was produced (not an empty/give-up build) | `non_empty_files` |
| `python_syntax` | STATIC | every `*.py` parses (`ast.parse`); an undecodable/null-byte file FAILs, never crashes (harness-mod-7) | `python_files` |
| `json_valid` | STATIC | every `*.json` is valid JSON; undecodable files FAIL cleanly | — |
| `image_loadable` | STRUCTURAL | every image decodes (`PIL.verify`) — no truncated/mislabelled art | `image_count` |
| `image_min_resolution` | STRUCTURAL | every image's short side ≥ 32px | `image_min_dim` |
| `image_not_blank` | DYNAMIC | every image has real variance (grayscale stddev > 2.0) — no flat fill | `image_min_stddev` |

The image checks certify only where Pillow is importable; a host without Pillow runs a smaller
certified suite (the CV checks stay inert, never waving art through).

## One Pond game checks (`build_onepond_registry`)

All read the committed `onepond_config.json` and gate a real game failure mode. Several are
**opt-in**: scoped to ponds that build the relevant piece, so earlier build-up tickets SKIP.

| id | tier | scope | gates | metric → floor |
|----|------|-------|-------|-----------|
| `onepond_placement_valid` | STRUCTURAL | all | legal layout: known types, in-bounds, no overlap, affordable | `onepond_buildings` |
| `auto_cohesion_check` | STRUCTURAL | all (≥2 buildings) | compactness (buildings ÷ bounding-box cells) ≥ 0.5 — not sprawl. **Harvested from a Stage-C proposal**; threshold later tightened 0.25→0.5 on a `tighten_rubric` signal | `onepond_cohesion` |
| `onepond_water_access` | STRUCTURAL | ponds with a `well` | every hatchery within Manhattan radius 2 of a well — no dry hatchery | `onepond_watered_hatcheries` |
| `onepond_economy_solvent` | DYNAMIC | all | bread never goes negative over 20 ticks | `onepond_min_bread`, `onepond_net_bread_delta` |
| `onepond_launch_viable` | DYNAMIC | ponds with a `launchpad` | at least one goose reaches space in 20 ticks — no dead launch infra | `onepond_launched` |
| `onepond_liveliness` | DYNAMIC | ponds with a `granary` | the flock is alive: ≥1 goose hatched — a granary's capacity isn't for a phantom flock | `onepond_geese_hatched` |
| `onepond_predator_safe` | DYNAMIC | ponds with `predators` + a hatchery | the flock survives the foxes (fences neutralize predators one-for-one) | `onepond_geese_protected` |

## The flywheel (Stage C)

Checks are not only hand-written. After a run, Stage C (`harness/review/decision_log_review.py`,
wired in `AutonomousRunner.harvest_stage_c`) mines the builders' decision logs for **recurring
subjective Stage-B defects** and proposes an adjustment for each:

- **`new_check`** — the criterion has no deterministic gate yet (suggested id `auto_<criterion>_check`).
  `auto_cohesion_check` above was born this way.
- **`tighten_rubric`** — the criterion *is already a certified check*, yet reviewers keep
  rejecting: the gate is too lax, tighten it (harness-mod-6). The `auto_cohesion_check`
  threshold bump demonstrated acting on this half.

Proposals are persisted to the `RunStore` and shown on the dashboard and autopilot summary.
